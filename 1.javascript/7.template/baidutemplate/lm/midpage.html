PC中间页

版本

目前PC中间页有三个版本，分别为tpclicked3hc，tpclicked3zx和tpclicked3_new

页面构成

中间页按照请求与展现逻辑划分，分成若干个区域，包括头尾、凤巢asp、闪投asp、问答asp、知心（游戏、教育和健康）、精品、个性和普适

tpclicked3_hc是将各个区域的代码写在了一起，代码在page.html中

tpclicked3zx和tpclicked3new通过codefiles的方式组织，实现了代码级别的复用

中间页对象

与检索的依赖数据被封装在uiParam对象上，代码在codefiles下midpage_sdk的uiParamnovaasp

业务逻辑通过midpage对象来完成，包括请求，展现以及交互的事件 内部通过cpro、zhixin和widget三个对象来实现具体逻辑，其中前两个是与广告检索相关

通过调用midpage的register方法，通过设置模版ID和预埋点，还可以调用midpage的runModule来实现类似功能，和前者的区别是前者会调用display方法

本质上，每个页面的区域对象都是继承自cpro、zhixin和widget三个对象中的一个，通过type属性表明具体继承自哪个对象

模版代码是被包裹在script标签中，采用类似php的方式编写，HTML、CSS和JavaScript混合，支持循环、分支等

页面渲染流程

使用midpage的register方法注册JSON对象后，

对于cpro和zhixin类型的对象，首先调用display方法，返回为真后会依次调用

init -> setCproAdParam -> beforeRequest -> setTemplateParam -> initAnti -> initEvent

以上的方法都有默认实现，也可以覆盖，但调用顺序不可以修改

对于widget类型的对象，首先调用display方法，返回为真后会调用

init -> setTemplateParam -> initAnti -> initEvent

模版

使用百度公共库baidu.template，参考链接为http://tangram.baidu.com/BaiduTemplate/

需要注意的问题

  1. 模版中style标签不能放在模版代码的第一行，因为IE下InnerHTML存在bug，第一行为style时不能通过InnerHTML赋值
  2. codefiles文件的每个key的value不能过长，过长后可能出现截断，这个时候需要使用codefiles的嵌套功能
  3. a标签的id必须保持唯一，推荐用prefix+xx+下标的做法，下标需要从0开始
